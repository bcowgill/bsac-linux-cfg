    Inline styling elements

    <b>A bold lead sentence in the paragraph, of course. When you need to <span>style some text as a group the span comes in handy. There are many other ways to make your text look. It can be very <strong> strong, if you like. And remember bold is bold but strong can do many things so long as it makes the text stronger. Sometimes you need to <sup> rise above it all and other times you're <sub> down in the dumps. When you just need to <u> draw a line under it all and move on, that is possible. You can <mark> mark some text with a highlight to indicate search matches or other relevance. Marking a passage as a direct quote without breaking up the text is <q> as good as it gets. You can indicate that a mistake was made by <s> striking text through. Closely related are the proofreading or differencing elements to show <del>deleted removed and <ins> inserted text. If you want to <small> speak more softly or try and hide what you are doing it's possible.

    When you need to <a>link to something else an anchor helps out. Visited, Unvisited and hovered links can look differently. Of course you can abbreviate things, <abbr>AFAIK, and a tooltip will spell it out for you.

    The <dfn>dfn element is used to mark the source of a definition and link a definition item back to it.

    When you absolutely have to have a line break
    <br> somewhere you can do that too. But other times you simply need to say where a word break is permitted if needed. Supercalifragilisticexpialidocious can be broken but iamunbreakablewillneverbesplitupnomatterhowwidethewindowis.

    Mentioning an artistic work which you can't connect to directly on the web is easy,<cite>Journal of the IEEE, 1979. You can progressively apply more and more stress emphasis to text by nesting an <em>em. The best way to show when to use the <i> and <em> elements are by example. <em> is about voice stress when you read it, but <i> is about titles, names and definitions. The Queen Mary sailed. Just do it already! The word the is an article. We had to do something about it.

    Telling someone about commands and programming code inline is also quite simple. <code>ls -alrt. You can indicate what the computer would respond back with <samp> file not found. As is signifying what should be typed on the keyboard <kbd> Ctrl-x Ctrl-s. <samp> Keyboard markup inside samples are a whole 'nother matter. <kbd> Esc. As are samples when they are within <kbd>keyboard sequences <samp>sample output. And to top it off a <kbd>keyboard sequence <kbd>within another one means something else entirely. Mathematical expressions or programs can have variables indicated. f(x) = ax + b. The variable width will set the button width.

    Associating some machine readable information is possible for Firefox users. We can tell the machine the ISBN number for <data>To Kill A Mockingbird. Time information can be shown in human format with a machine readable annotation. The concert took place on <time> May 15. Lights out are at 22:00.
