Essential Reference for the Linux Bash Terminal, Commands, and Scripting

Use this guide to learn how to use the Linux Bash Terminal, Commands and Scripting for navigation, editing, command control, and easy access to history. We will get you looking like a pro in no time at all.

The Linux Bash (Bourne Again Shell) is a shell and command language that is known to most Linux users, since it's the default shell in most Linux distributions.

A beginner Linux user may find the Linux terminal usage a bit hard to grasp, especially because it relies heavily on using your arrow keys to move around. Additionally, constantly re-typing the commands, each with a slight variation, can be annoying.

However, Bash comes with many easy-to-use keyboard shortcuts and history functions that ensure efficient use of the terminal. We will introduce them here in stages to guide you to mastery.

Help, get me out of here!

First, let's show you how to get out of things when you get stuck.

Key / Command

Name / Mnemonic

Action

Esc

Escape

Escape from the current mode of a program, or from the program itself. Sometimes you will need more than one to do the job.

Ctrl-C

Break

Terminate the currently running program. Sends SIGINT signal and kills currently executing command.

Ctrl-D

End of input

Terminates standard input if the program is waiting for you. If not, closes the current terminal.

So you might want to start a second shell to avoid getting logged out. Just type bash at the command prompt to start another shell level. Type echo $SHLVL to see how deep your shell rabbit hole goes. When $SHLVL is 1 you are at risk of logging out when Ctrl-D is pressed next.

Ctrl-Z

Snooze

Puts the current commnd to sleep so you can do something else.

You can see how many such stopped jobs you have with the jobs command. You can resume a stopped job with the fg command to bring it back into the foreground so you can control it again. You can resume a job in the background with the bg command. It will run happily without you but will be suspended again if it needs to ask you for any information. Use the job number from the jobs command if you have more than one stopped job i.e. bg 3

Ctrl-S

Pause

Stops the command from showing output to the screen. It keeps chugging away at it, but you can't see it.

Ctrl-Q

Continue

Resumes showing command output to the screen. Picks up where you paused it so you don't miss anything.

Ctrl-L

Clear

Similar to clear command, clears the terminal screen.

ps

Processes

Shows what processes (programs) and stopped jobs are running.

You can end any of them using the number under the PID column with the kill -HUP PID command. If it doesn't work the first time, change -HUP to -TERM, -SIGINT or -KILL until it does.

man command-name

Manual Page

Help on commands from the instruction manual. When finished reading, press q to quit.

Use man bash to get help on the shell itself (4000+ lines of wisdom there.)

Getting around the command line.

Second, how to get around the command line like a pro! Here is the anatomy of a command:

ls  -alrt /tmp /etc/*.sh "my file.txt" <Enter>
[A] [ B ] [C ] [-- D --] [---- E ----] [- F -]

A = The command or program name. Everything after the command name are called parameters or arguments.

B = Options, switches, flags, settings which affect the command. Usually begin with - or -- and can be separated as -a -l or combined as -al Sometimes they require a value as -c3, -c=3 or -c 3

C, D and E are directory, path, or filename parameters. You must be careful to use quotation marks to surround any parameters with spaces in them or they will be mistaken for separate values like my and file.txt instead of "my file.txt"

F = End of command. You can also use other ways to group or end the command on one line and maybe put another on the same line i.e & ; && || # | < > 2> >> 2>> ( ) ` ` $( ) \

Key / Command

Name / Mnemonic

Action

Ctrl-A

Home

Move to the start of the command line

Ctrl-E

End

Move to the end of the command line

history

Show History

Shows full history of bash commands which have been executed.

You can filter the list to find matches with history | grep command. Then you can re-run a numbered command using the original parameters with !number or you can run the previous command starting with 'perl' as !perl

Ctrl-R command

History Match

Reverse search the bash command history.

As you type the command text it will search and show you the matching command. Press Ctrl-R again to get the previous one. When you find the one you want, press Tab, Esc, or Ctrl-J to accept it for further editing before it is executed. Or press Ctrl-C to abort and start over.

^match^replace

History Replace

Re-run the previous command after a replacement.

If you made a mistake in the previous command line you can quickly correct it without retyping or a lot of arrow key editing.

It is also a safe way to see what files will be deleted before you remove them. For example do ls /tmp/*.sh to see what files would match, then if you are happy to delete all of them you can just say ^ls^rm and the command executed will be rm /tmp/*.sh

!number:s/match/replace

History Replace

Re-run the numbered command after a search and replacement.

After a history list show you are a real smartass by re-running a numbered command with a typo error correction. i.e. !42:s/fnid/find/

To master history see the HISTORY EXPANSION section of the man bash page.
